Introduction

Build, scale, and manage user-facing Retrieval-Augmented Generation applications.

R2R (RAG to Riches), the Supabase for RAG, bridges the gap between experimenting with and deploying production-ready Retrieval-Augmented Generation (RAG) applications. It’s a complete platform that helps you quickly build and launch scalable RAG solutions. R2R is built around a simple RESTful API, making it easy to use and fast to implement.

​
Key Features

  • 📁 Multimodal Support: Ingest files ranging from .txt, .pdf, .json to .png, .mp3, and more.
  • 🔍 Hybrid Search: Combine semantic and keyword search with reciprocal rank fusion for enhanced relevancy.
  • 🔗 Graph RAG: Automatically extract relationships and build knowledge graphs.
  • 👤 User Management: Efficiently manage users, documents, and permissions with full authentication.
  • 🔭 Observability: Observe and analyze your RAG engine performance.
  • 🔌 Extensibility: Develop your application further with easy builder + factory pattern.
  • 🖥️ WebApp: Interact with and manage R2R with the R2R Application, an open-source React+Next.js front-end with integrated authentication for user-facing applications.

​
Quickstart

After installing, the R2R Quickstart is your go to for a step-by-step guide to get up and running with R2R in minutes. The guide demonstrates R2R’s Retrieval-Augmented Generation (RAG) system by ingesting sample documents and then showcasing features for search, RAG, logging, analytics, and document management.

To get started with R2R, we recommend starting with the quickstart and then moving on to specific cookbooks.

  • Installation: Quick setup using Docker or pip
